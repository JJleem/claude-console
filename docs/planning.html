프로젝트 개요

Claude Console은 Claude Code를 사용하는 개발자가 로컬 환경에서 LLM 호출 로그를 수집·분석하고, 프롬프트를 체계적으로 관리하며, 메모리와 Eval 파이프라인을 직접 운영할 수 있는 올인원 관찰성(Observability) 도구입니다. 외부 SaaS에 의존하지 않고 100% 로컬에서 동작하며, npm run dev 단 하나의 명령으로 실행됩니다.

  • Prompt Harness로 프롬프트 버전관리 + A/B 테스트

기능 명세

📊
Overview
API 호출 통계 · 비용 · 토큰 요약

Claude Code가 생성한 모든 API 호출을 집계하여 사용량, 비용, 토큰 소비를 한눈에 파악합니다. 일별/모델별 트렌드 차트와 실시간 지표를 제공합니다.

  • 총 API 호출 수 / 비용 / 입력·출력 토큰 카드 — 오늘, 7일, 전체 기준

  • 모델별 분포 파이/도넛 차트 — Sonnet vs Haiku 비율
  • 평균 응답 지연, P95 latency 표시
  • 에러율 지표 — 실패한 API 호출 비율
SQLite aggregation Prisma ORM Chart.js / Recharts

Runs
LLM 호출 로그 전수 기록

Claude API로 보내는 모든 요청·응답을 캡처하여 저장합니다. 각 Run에는 프롬프트 내용, 응답, 토큰, 비용, 지연시간, 에러 정보가 포함됩니다. 구현 우선순위 1번으로, 다른 모든 기능의 데이터 기반이 됩니다.

  • LLM API Wrapper — Claude API 호출을 가로채서 SQLite에 기록
  • Run 상세 뷰 — 전체 메시지 히스토리, system/user/assistant 구분 렌더링
  • Vision 지원 — 이미지가 포함된 멀티모달 호출 이미지 표시
  • 시맨틱 검색 — sqlite-vec 임베딩으로 내용 기반 Run 검색 (RAG)
  • 필터링 — 모델, 날짜, 비용 범위, 에러 여부
LLM API Wrapper SQLite + Prisma sqlite-vec Multimodal Vision
🤖
Agents
멀티 에이전트 실행 흐름 시각화

Claude Code의 서브에이전트 실행 트리를 시각화합니다. 어떤 에이전트가 어떤 에이전트를 호출했는지, 각 단계의 소요 시간과 토큰 사용량을 추적합니다.

  • Agent Tree 뷰 — 부모-자식 에이전트 관계를 계층형 트리로 렌더링
  • 타임라인 뷰 — 병렬 에이전트 실행 구간을 Gantt 스타일로 표시
  • 노드 상세 — 각 에이전트의 입출력, 사용 도구, 에러 상태
  • 실행 통계 — 총 소요시간, 병렬도, 비용 분포
Multi-agent Tracing Tree Visualization Timeline / Gantt
📝
Prompts
CLAUDE.md 에디터 + Prompt Harness

CLAUDE.md와 커스텀 프롬프트를 버전 관리하고, A/B 비교 실험을 실행합니다. Prompt Harness를 통해 동일한 입력에 대한 다양한 프롬프트 버전의 출력을 나란히 비교합니다.

  • CLAUDE.md 에디터 — 마크다운 에디터로 실시간 편집 + 저장 + 토큰 카운트
  • 버전 히스토리 — Git-style diff로 프롬프트 변경 이력 추적
  • Prompt Studio (A/B) — 두 프롬프트 버전을 동시에 실행해 출력 비교
  • Context Viewer — 실제 API로 전송되는 전체 컨텍스트 + 토큰 미터
  • 템플릿 라이브러리 — 재사용 가능한 프롬프트 스니펫 저장
Prompt Harness A/B Testing Context Engineering Version Control

Hooks
Hook 이벤트 시각화 + CRUD

Claude Code의 PreToolUse / PostToolUse / Stop Hook 이벤트를 캡처하여 시각화하고, 새로운 Hook 스크립트를 GUI로 생성·편집·삭제합니다.

  • Hook 이벤트 로그 — 발생한 Hook 이벤트 타임라인 및 페이로드 표시
  • Hook CRUD 에디터 — 새 Hook 스크립트 작성, 활성화/비활성화 토글
  • 실행 결과 추적 — Hook이 반환한 값, 오류, 실행 시간
  • Hook 타입 필터 — PreToolUse / PostToolUse / Stop 구분 뷰
Claude Code Hooks Event Visualization Script CRUD

  • Skill 목록 — 모든 슬래시 커맨드와 설명, 최근 사용일 표시

Memory
memory/ 파일 카드뷰 + 시맨틱 검색 (RAG)

Claude Code의 memory/ 디렉토리 파일들을 카드 형식으로 브라우징하고, sqlite-vec 기반 임베딩 검색으로 의미론적으로 유사한 기억을 찾습니다. 구현 우선순위 2번 — RAG 파이프라인의 핵심입니다.

  • Memory 카드 뷰 — 각 memory 파일을 카드로 렌더링, 빠른 탐색
  • 시맨틱 검색 — 쿼리 임베딩 → sqlite-vec ANN 검색 → 관련 메모리 반환
  • 임베딩 파이프라인 — 파일 변경 감지 → 자동 임베딩 재생성 → 저장
  • 유사도 스코어 — 검색 결과에 코사인 유사도 시각화
  • 메모리 편집 — GUI에서 직접 memory 파일 내용 수정
RAG Pipeline sqlite-vec Embedding Semantic Search ANN Search

settings.json MCP Servers Model Config

Eval
LLM-as-judge 자동 품질 평가

저장된 Run 로그를 대상으로 LLM-as-judge 방식의 자동 평가를 실행합니다. Eval 에이전트가 Tool Use로 Run 데이터를 조회하고, 품질 기준에 따라 점수를 매깁니다. 구현 우선순위 3번 — Tool Use 학습의 핵심입니다.

  • Eval Runner — Run을 선택하면 judge LLM이 자동으로 품질 평가 실행
  • Tool Use 에이전트 — Eval 에이전트가 get_run, search_runs 툴로 데이터 조회
  • 평가 기준 설정 — 정확성, 간결성, 유해성 등 기준 커스터마이징
  • 점수 대시보드 — Run별 Eval 점수 이력 및 트렌드
  • Prompt A/B Eval — 두 프롬프트 버전의 Eval 점수를 나란히 비교
LLM-as-judge Tool Use Function Calling Auto Scoring

  • 실시간 토큰 스트림 — 현재 생성 중인 응답을 실시간 타이핑 효과로 표시
  • 활성 에이전트 패널 — 현재 실행 중인 에이전트 목록 및 상태
  • 스트림 토글 — 실시간 수신 일시정지 / 재개
  • 이벤트 로그 — 스트리밍 이벤트 타입별 색상 구분 표시

Memory 시맨틱 검색 (RAG)

진행 방식

1
기반 설계 (Day 1~2)
Next.js 프로젝트 초기화 + Prisma 스키마 설계 + SQLite 마이그레이션. LLMCallRecord, EvalResult, MemoryChunk 등 핵심 테이블 정의. 사이드바 레이아웃 + 라우팅 구조 완성.
2
Runs 로깅 구현 (Day 2~3)
Anthropic SDK를 래핑하는 loggedAnthropicCall() 유틸리티 작성. 모든 요청/응답 메타데이터를 SQLite에 저장. Runs 목록 + 상세 UI 구현. Overview 통계 집계 쿼리 작성.

RAG 파이프라인 구축 (Day 3~5)
